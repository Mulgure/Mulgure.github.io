for-in

for-in 遍历属性的顺序并不确定，即输出的结果顺序与属性在对象中的顺序无关，也与属性的字母顺序无关，与其他任何顺序也无关。(似乎是跟浏览器有关 其实绝大部分情况还是按输入顺序输出的 旧式浏览器就难说了)

for-in 不仅仅遍历 array 自身的属性，其还遍历 array 原型链上的所有可枚举的属性。

eg:

Array.prototype.fatherName = "Father";
const arr = [1, 2, 3];
arr.name = "Hello world";
let index;
for(index in arr) {
    console.log("arr[" + index + "] = " + arr[index]);
}

结果:

arr[0] = 1
arr[1] = 2
arr[2] = 3
arr[name] = Hello world
arr[fatherName] = Father

ps:for-in性能不佳 除非明确需要迭代一个属性数量未知的对象 否则应避免使用for-in

export interface StringValidator {
    isAcceptable(s: string): boolean;
}

ZipCodeValidator.ts

export const numberRegexp = /^[0-9]+$/;

export class ZipCodeValidator implements StringValidator {
    isAcceptable(s: string) {
        return s.length === 5 && numberRegexp.test(s);
    }
}

export { ZipCodeValidator };

let myAdd: (x:number, y:number)=>number =

=>左边：参数; 右边：返回值

可选参数和默认参数:

传递给一个函数的参数个数必须与函数期望的参数个数一致。

    function buildName(firstName: string, lastName: string) {
    return firstName + " " + lastName;
 }

    let result1 = buildName("Bob");                  // error, too few parameters
    let result2 = buildName("Bob", "Adams", "Sr.");  // error, too many parameters
    let result3 = buildName("Bob", "Adams");         // ah, just right

可选参数：加"?"即可，跟接口的可选属性一样.PS:可选参数必须跟在必须参数后面

    function buildName(firstName: string, lastName?: string) {
   ...
}

如果带默认值的参数出现在必须参数前面，用户必须明确的传入 undefined值来获得默认值。
    function buildName(firstName = "Will", lastName: string) {
        return firstName + " " + lastName;
}
    let result4 = buildName(undefined, "Adams");     // okay and returns "Will Adams"

关于this

JavaScript里，this的值在函数被调用的时候才会指定。 

然而，箭头函数能保存函数创建时的 this值，而不是调用时的值。( lambda表达式 () => {}   )

也就是说，在箭头函数中,可以理解为 var _this = this;然后把this替换成_this

class Animal {

接口(1) (好像有点难啊，不知道到时候还会不会这么想)

用interface定义 interface+接口名 {属性1；属性2；}

eg: 

interface LabelledValue {
    label: string;
}

label: string –必选属性

还有：

label?: string –可选属性(在属性名后加问号） —-不是很清楚这玩意到底干嘛的，文档没看懂
readonly label: string –只读属性(在属性名前加 readonly) 只能在对象刚刚创建的时候修改其值

eg：

interface Point {
    readonly x: number;
    readonly y: number;
}
let p1: Point = { x: 10, y: 20 };
p1.x = 5; // error!x,y不能再改变

乱七八糟的：接口的属性检查(乱死了，真的不是很懂这玩意，只能照搬文档代码了)

 interface SquareConfig {
    color?: string;
    width?: number;
}

function createSquare(config: SquareConfig): { color: string; area: number } {
    // ...
}

let mySquare = createSquare({ colour: "red", width: 100 });//这里的colour跟color是不一样，然后就有error: 'colour' not expected in type 'SquareConfig'

绕开的方法：

1.类型断言 let mySquare = createSquare({ width: 100, opacity: 0.5 } as SquareConfig);
2.添加一个字符串索引签名 如果 SquareConfig带有上面定义的类型的color和width属性，并且还会带有任意数量的其它属性，那么我们可以这样定义它
   interface SquareConfig {
    color?: string;
    width?: number;
    [propName: string]: any;
    }
3.将这个对象赋值给一个另一个变量： 因为 squareOptions不会经过额外属性检查，所以编译器不会报错。
let squareOptions = { colour: "red", width: 100 };
let mySquare = createSquare(squareOptions);
